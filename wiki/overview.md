# LLM Wiki OpenCode

This project is a meta-implementation of the [LLM Wiki pattern](./concepts/llm-wiki-pattern.md) — an OpenCode framework-based AI agent system that autonomously builds and maintains a persistent wiki from source documents. Rather than treating the project as merely a tool to be used, the project itself demonstrates the pattern it describes: the wiki you are reading was generated by the very system it documents.

## What Is the LLM Wiki Pattern?

Most interactions with LLMs and documents follow a retrieval-augmented generation (RAG) approach: upload files, the LLM retrieves relevant chunks at query time, and generates an answer. This model is inefficient because the LLM rediscovers knowledge from scratch on every question. Nothing accumulates.

The LLM Wiki pattern inverts this. Instead of retrieving from raw documents at query time, the LLM incrementally builds and maintains a structured wiki that sits between you and the raw sources. When new sources are added, the LLM integrates the information into the existing wiki — updating entity pages, revising topic summaries, noting contradictions, and strengthening cross-references. The knowledge is compiled once and kept current, not re-derived on every query.

The key difference: **the wiki is a persistent, compounding artifact.** Cross-references are already established. Contradictions are flagged. Synthesis reflects everything read. The wiki grows richer with every source added and every question asked.

## Three-Layer Architecture

The architecture is documented in detail in [Architecture](./architecture.md).

The pattern operates across three distinct layers:

**Raw Sources** — A curated collection of immutable source documents. Articles, papers, images, data files. The LLM reads from these but never modifies them. They serve as the source of truth.

**The Wiki** — A directory of LLM-generated markdown files. Summaries, entity pages, concept pages, comparisons, and overviews. The LLM owns this layer entirely — creating pages, updating them as new sources arrive, maintaining cross-references, and keeping everything consistent.

**The Schema** — A configuration file that instructs the LLM on wiki structure, conventions, and workflows for ingesting sources, answering queries, and maintaining the wiki. This is the key configuration that makes the LLM a disciplined wiki maintainer rather than a generic chatbot.

## Key Features

- **Autonomous wiki maintenance** — The LLM handles all summarizing, cross-referencing, filing, and bookkeeping that makes a knowledge base useful over time
- **Persistent compounding knowledge** — Unlike RAG systems, the wiki accumulates information rather than re-deriving it on each query
- **Multi-source integration** — A single source ingestion can touch 10-15 wiki pages, updating entity and concept pages across the wiki
- **Exploration-to-artifact workflow** — Answers to questions can be filed back into the wiki as new pages, so explorations compound in the knowledge base
- **Periodic health-checking** — The lint operation checks for contradictions, stale claims, orphan pages, and missing cross-references
- **Version-controlled content** — The wiki is a git repository of markdown files, providing version history and branching for free

## Why This Approach Works

The tedious part of maintaining a knowledge base is not the reading or the thinking — it is the bookkeeping. Updating cross-references, keeping summaries current, noting when new data contradicts old claims. Humans abandon wikis because the maintenance burden grows faster than the value. LLMs do not get bored, do not forget to update a cross-reference, and can modify 15 files in one pass. The wiki stays maintained because the cost of maintenance is near zero.

The human's role is to curate sources, direct the analysis, ask good questions, and think about what it all means. The LLM's role is everything else.

## Operations

The wiki supports three primary operations:

**Ingest** — Drop a new source into the raw collection and the LLM processes it. The LLM reads the source, discusses key takeaways, writes a summary page in the wiki, updates the index, updates relevant entity and concept pages, and appends an entry to the log.

**Query** — Ask questions against the wiki. The LLM searches for relevant pages, reads them, and synthesizes an answer with citations. Answers can take various forms — a markdown page, a comparison table, a slide deck, a chart. Valuable answers can be filed back into the wiki as new pages.

**Lint** — Periodically health-check the wiki. Look for contradictions between pages, stale claims superseded by newer sources, orphan pages with no inbound links, and important concepts lacking their own page.
